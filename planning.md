### Tool 1: search_listings

**What it does:**
Searches the mock listings dataset for items matching a natural-language description, with optional filters for size and maximum price. Results are ranked by keyword relevance and returned in descending order of match quality.

**Input parameters:**

* `description` (str): Keywords describing what the user wants (e.g., `"vintage graphic tee"`). Used for keyword scoring against the `title`, `description`, `style_tags`, and `colors` fields.
* `size` (str | None): Size filter, or `None` to skip filtering. Case-insensitive partial matching is used (e.g., `"M"` matches `"S/M"` and `"M/L"`).
* `max_price` (float | None): Maximum allowed price (inclusive), or `None` to skip price filtering.

**What it returns:**
A list of listing dictionaries sorted by relevance score (highest first). Each listing contains:

* `id`
* `title`
* `description`
* `category`
* `style_tags` (list[str])
* `size`
* `condition`
* `price` (float)
* `colors` (list[str])
* `brand` (str | None)
* `platform`

Returns an empty list (`[]`) if no matching listings are found.

**What happens if it fails or returns nothing:**
The agent sets `session["error"]` to a helpful message such as:

> "No listings found for '[query]'. Try broadening your search — remove the size filter, raise the price ceiling, or use different keywords."

The agent then returns the session immediately without calling `suggest_outfit` or `create_fit_card`.

---

### Tool 2: suggest_outfit

**What it does:**
Generates 1–2 outfit suggestions that incorporate a thrifted item the user is considering purchasing along with pieces from their existing wardrobe. Uses an LLM to create personalized styling recommendations.

**Input parameters:**

* `new_item` (dict): A listing dictionary returned by `search_listings`.
* `wardrobe` (dict): A wardrobe dictionary containing an `items` key with a list of wardrobe item dictionaries. May be empty.

**What it returns:**
A non-empty string containing 1–2 outfit suggestions. When wardrobe items are available, the suggestions reference specific pieces by name. If the wardrobe is empty, the tool returns general styling advice describing what types of items pair well with the thrifted piece and the overall aesthetic it suits.

**What happens if it fails or returns nothing:**
If the LLM call fails, the tool returns a fallback message such as:

> "Couldn't generate outfit suggestions right now, but this [item category] would pair well with neutral basics."

The tool never raises an exception and never returns an empty string.

---

### Tool 3: create_fit_card

**What it does:**
Creates a short, shareable outfit caption inspired by Instagram or TikTok OOTD posts. The caption is generated using the outfit suggestion and the thrifted item's details to produce a casual, authentic style summary.

**Input parameters:**

* `outfit` (str): The outfit suggestion generated by `suggest_outfit`.
* `new_item` (dict): The thrifted item listing, including details such as `title`, `price`, and `platform`.

**What it returns:**
A 2–4 sentence caption that:

* Sounds casual and authentic
* Mentions the item's name once
* Mentions the item's price once
* Mentions the platform once
* Captures the overall vibe of the outfit

The tool uses a higher LLM temperature (0.9+) to ensure varied captions across calls. If the outfit input is invalid, it returns a descriptive error message instead of raising an exception.

**What happens if it fails or returns nothing:**

* If `outfit` is empty or contains only whitespace, the tool returns:

> "Error: outfit description is required to generate a fit card."

* If the LLM call fails, the tool returns a fallback caption generated from the item's details alone. The tool never raises an exception.

## Planning Loop

**How does your agent decide which tool to call next?**
The agent works through a fixed sequence of steps, but branches early based on tool results:

Parse the user's query string using the LLM to extract description, size, and max_price. Store in session["parsed"].
Call search_listings(description, size, max_price). Store results in session["search_results"].

Branch: If results is empty → set session["error"] to a helpful message, return session early. Do NOT call suggest_outfit.

Set session["selected_item"] = results[0] (top-ranked result).
Call suggest_outfit(selected_item, wardrobe). Store in session["outfit_suggestion"].
Call create_fit_card(outfit_suggestion, selected_item). Store in session["fit_card"].
Return session.


The key conditional is step 2: the agent's behavior changes based on whether search returns results. Only the happy path reaches steps 4–6.
---

## State Management

**How does information from one tool get passed to the next?**
All state lives in a session dict initialized by _new_session(). Fields:


query: original user query string
parsed: dict with keys description, size, max_price extracted from query
search_results: list of listing dicts from search_listings
selected_item: the top result dict, passed into suggest_outfit
wardrobe: the user's wardrobe dict, passed into suggest_outfit
outfit_suggestion: string returned by suggest_outfit, passed into create_fit_card
fit_card: string returned by create_fit_card
error: None on success; set to a message string if the interaction terminates early


State flows forward: selected_item (from search) → suggest_outfit → outfit_suggestion → create_fit_card. No tool re-prompts the user or re-reads the query.

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool              | Failure mode                          | Agent response            |
| ----------------- | ------------------------------------- | ------------------------- |
| `search_listings` | No results match the query            | Sets `session["error"]` to: *"No listings found for '[query]'. Try broader keywords, remove the size filter, or raise your price ceiling."* The agent then returns early without calling `suggest_outfit` or `create_fit_card`. |

| `suggest_outfit`  | Wardrobe is empty                     | Generates general styling advice instead of wardrobe-specific recommendations. Returns a non-empty string such as: *"This piece pairs well with high-waisted jeans, straight-leg trousers, or layered basics."*      
           |
| `create_fit_card` | Outfit input is missing or incomplete | Returns the error message: *"Error: outfit description is required to generate a fit card."* and does not attempt to call the LLM.                                                                                              |

    │
    ├─── Step 4: create_fit_card(outfit_suggestion, selected_item)
    │              │
    │              └──► session["fit_card"] = caption str
    │
    └──► return session

## A Complete Interaction (Step by Step)

Write out what a full user interaction looks like from start to finish — tool call by tool call. Use a specific example query.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The agent calls `search_listings(description="vintage graphic tee", size=None, max_price=30.0)`. The function loads all listings, filters out items priced above $30, scores the remaining listings based on keyword overlap with "vintage graphic tee" (using the title, description, and style_tags fields), and returns the results sorted by relevance. The top result is **"Graphic Tee — 2003 Tour Bootleg Style"** priced at **$24** on **Depop**.

**Step 2:**
The agent stores the selected listing in `session["selected_item"]` and calls `suggest_outfit(new_item=<band tee dict>, wardrobe=<example_wardrobe>)`. The LLM receives the thrifted item's details and the user's wardrobe items, then generates an outfit suggestion such as: *"Pair this boxy black graphic tee with your baggy straight-leg jeans and chunky white sneakers for a classic 90s streetwear look. Add your black crossbody and you're done."*

**Step 3:**
The agent calls `create_fit_card(outfit="Pair this boxy...", new_item=<band tee dict>)`. Using the outfit recommendation and item details, the LLM generates a short social-style caption such as: *"found this 2003 bootleg tee on depop for $24 and it was made for my baggy jeans 🖤 throwing on the chunky sneakers and calling it an outfit honestly"*

**Final output to user:**

* **Panel 1 (Listing):** Title, price, platform, condition, size, and style tags of the recommended thrifted item.
* **Panel 2 (Outfit Idea):** The complete outfit recommendation generated by `suggest_outfit`.
* **Panel 3 (Fit Card):** The shareable outfit caption generated by `create_fit_card`.
